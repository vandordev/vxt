# v0.1.0

## Summary

- first public Go package release of `vxt`
- experimental `v0.x` release with an intentionally small public API

## Supported Public Packages

- `github.com/vandordev/vxt`
- `github.com/vandordev/vxt/runtime`
- `github.com/vandordev/vxt/bind`
- `github.com/vandordev/vxt/diag`
- `github.com/vandordev/vxt/source`
- `github.com/vandordev/vxt/write`

Implementation packages under `internal/` are not part of the supported public
API.

## Highlights

- single-file rendering through `vxt.RenderSingleFile`
- document pipeline for compile, validate, plan, and write
- structured diagnostics for compile, validate, plan, and write stages
- filesystem and memory output targets
- explicit post-write `ApplyPlan` hook execution through an injected executor
- document-mode generated Go bindings through `bind`

## Non-Goals

- CLI distribution
- automatic hook execution
- registry or package semantics
- `v1` API stability guarantees

## Verification

Release verification for `v0.1.0` should include:

```bash
go test ./...
go list ./...
go doc github.com/vandordev/vxt
go doc github.com/vandordev/vxt/runtime
go doc github.com/vandordev/vxt/bind
go doc github.com/vandordev/vxt/write
./scripts/release-smoke.sh
```

See [docs/releasing.md](../releasing.md) for the repeatable release workflow
and tagging checklist.

Before the public tag is resolvable remotely, `./scripts/release-smoke.sh` uses
`go mod edit -replace` to point `github.com/vandordev/vxt` at the local repo
checkout for pre-release verification.
