# VXT v0.1.0 Release Readiness Implementation Plan

> **For agentic workers:** REQUIRED: Use `superpowers:subagent-driven-development` (if subagents available) or `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare `github.com/vandordev/vxt` for a first public Go package release as `v0.1.0` with clear public docs, a repeatable external smoke test, and a documented release workflow.

**Architecture:** Keep release-readiness work in three focused units: public-consumer documentation, external-consumer verification, and release mechanics. Avoid runtime behavior changes unless a release verification step exposes a real packaging problem.

**Tech Stack:** Go modules, Markdown docs, shell scripting, existing `go test` and `go doc` tooling.

---

## Chunk 2: External Consumer Smoke Test

### Task 2: Add a repeatable public-consumer smoke test

**Files:**
- Create: `smoketest/publicapi/main.go`
- Create: `scripts/release-smoke.sh`
- Modify: `docs/releases/v0.1.0.md`
- Test: run `scripts/release-smoke.sh`

- [ ] **Step 1: Write the consumer fixture program first**

Create `smoketest/publicapi/main.go` as a tiny external-consumer example that:
- imports only supported public packages
- compiles a small document source
- validates input
- plans output
- writes to `write.NewMemoryTarget()`
- exits non-zero if diagnostics or write errors occur

Use a minimal inline document source similar to:

```go
src := source.Source{
	ID: "release-smoke.vxt",
	Text: "@template smoke\n" +
		"@input name string\n" +
		"@file \"hello.txt\"\n" +
		"Hello {{ name }}\n" +
		"@endfile\n",
}
```

- [ ] **Step 2: Run the fixture locally from inside the repo to catch API mistakes before writing the shell script**

Run:

```bash
go run ./smoketest/publicapi
```

Expected:
- program exits successfully
- output indicates exactly one file was planned or written in memory

- [ ] **Step 3: Create the external smoke script**

Create `scripts/release-smoke.sh` that:
- resolves the repo root
- creates a temporary directory outside the repo
- initializes a fresh Go module
- adds `github.com/vandordev/vxt`
- uses `go mod edit -replace github.com/vandordev/vxt=<repo-root>` for pre-tag local verification
- copies or writes the smoke fixture program into the temp module
- runs `go run .`
- cleans up the temp directory on exit

The script should use strict shell settings:

```bash
#!/usr/bin/env bash
set -euo pipefail
```

- [ ] **Step 4: Run the smoke script and make it pass**

Run:

```bash
./scripts/release-smoke.sh
```

Expected:
- a fresh external module is created
- the module resolves `github.com/vandordev/vxt` through the local replace directive
- `go run .` succeeds
- the script exits `0`

- [ ] **Step 5: Document the smoke test in the release notes**

Update `docs/releases/v0.1.0.md` with a `## Verification` section that includes:
- `go test ./...`
- `go list ./...`
- `go doc ...`
- `./scripts/release-smoke.sh`

- [ ] **Step 6: Commit the smoke-test unit**

Run:

```bash

git commit -m "test: add v0.1.0 public release smoke test"
```
