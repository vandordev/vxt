# VXT Bind Write Output Implementation Plan

> **For agentic workers:** REQUIRED: Use `superpowers:subagent-driven-development` (if subagents available) or `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `bind.WriteOutput(...)` and `bind.GenerateToDir(...)` so generated Go bindings can be written directly into an explicit output directory with safe scoped cleanup.

**Architecture:** Extend the existing `bind` package with output metadata, deterministic generated-file ownership markers, a narrow filesystem writer, and a convenience wrapper over `Generate`. Keep generation side-effect free and isolate all filesystem behavior into one small writer unit.

**Tech Stack:** Go, existing `bind` package, `os` and `path/filepath`, generated Go source headers, `go test`, temp-directory filesystem tests.

- [ ] **Step 1: Write failing stale-cleanup tests**

Extend `bind/write_test.go` with scenarios where the output directory already contains:
- an old owned file for the same binding that should be removed
- a generated file for another binding that must be preserved
- a random non-generated file that must be preserved

Example directory contents before write:

```text
.vxt/
  service_gen.go         // owned by service
  service_old_gen.go     // owned by service, stale
  repository_gen.go      // owned by repository
  notes.txt              // unrelated
```

New output only contains `service_gen.go`.

Expected after `WriteOutput`:
- `service_gen.go` present
- `service_old_gen.go` removed
- `repository_gen.go` preserved
- `notes.txt` preserved

- [ ] **Step 2: Run the stale-cleanup tests and confirm they fail**

Run:

```bash

```

Expected:
- FAIL because stale cleanup is not implemented

- [ ] **Step 3: Add ownership detection helpers**

Create `bind/ownership.go` with focused helpers such as:

```go
func ownedGeneratedFileName(bindingName string) string
func isOwnedBindingFile(bindingName, packageName, path, content string) bool
```

Rules:
- the filename must match the binding scope
- the header marker must match both binding and package

- [ ] **Step 4: Implement scoped reconcile**

Update `bind/write.go` so after writing the current output set it:
- scans only the explicit output directory
- identifies files owned by the same binding/package
- removes owned files not present in the new output set
- records removed files in `WriteReport.FilesRemoved`

Keep cleanup intentionally narrow:
- do not recurse into subdirectories
- do not touch files without ownership marker
- do not touch files owned by a different binding

- [ ] **Step 3: Implement the convenience wrapper**

Add to `bind/generator.go`:

```go
func GenerateToDir(req Request, dir string) (WriteReport, error) {
	out, err := Generate(req)
	if err != nil {
		return WriteReport{}, err
	}
	return WriteOutput(out, dir)
}
```
